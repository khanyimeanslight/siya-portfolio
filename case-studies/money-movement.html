Role: Lead Product Designer / Senior Product Design Manager

Team: Working with payments, engineering, fraud, and operations teams

The Challenge

Business Problem
User Problem
Money movement journeys with high reliability and high consequence of failure. When payments failed, customers received unclear error messages and didn't know what to do next.
Constraints
Reversibility limits, settlement timing, fraud controls, and operational interventions. Must balance speed, security, and customer clarity.

Discovery & Research

User Research

Exception Analysis

  • Mapped all payment states: initiation, authorization, processing, settlement, failure
  • Analyzed 1000+ exception cases to identify patterns
  • Identified recovery paths and alternative rails

Strategy & Approach

Design Principles

Transparency

Clear, honest communication about payment status at every step.

Recovery

Every failure includes a clear path forward.

Reliability

Design for resilience and graceful degradation.

Trust

Build confidence through consistent, accurate messaging.

State Model Design

  • Defined state model: initiation, authorization, processing, settlement, failure
  • Designed recovery paths: retry logic, alternative rails, and escalation triggers
  • Standardized message taxonomy for accuracy and compliance

Design & Iteration

Key Design Decisions

Real-Time Status Updates
Live payment status with clear explanations at each stage, reducing anxiety and support tickets.
Smart Error Messages
Specific, actionable error messages instead of generic failures, with immediate next steps.
Automatic Recovery
Intelligent retry logic with user notification, reducing manual intervention needs.
Exception Dashboard
Clear view of payment history and status, building trust through transparency.

Implementation & Handoff

  • Created comprehensive state machine documentation
  • Designed component library for payment status and error states
  • Collaborated with engineering on retry logic and recovery flows
  • Conducted user testing with real payment scenarios
