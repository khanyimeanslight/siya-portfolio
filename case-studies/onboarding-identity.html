Strategy & Approach

Design Principles

Established 5 core principles to guide all design decisions:

Trust

Build confidence through transparency and clear communication at every step.

Clarity

Make verification status and next steps immediately understandable.

Efficiency

Minimize steps while maintaining regulatory compliance.

Compliance

Ensure every interaction meets audit and regulatory requirements.

Accessibility

Design for all users, meeting WCAG AA standards.

Information Architecture

Redesigned the information architecture to support progressive disclosure and clear status communication:

  • Mapped end-to-end flow: user intent, control points, and data lineage
  • Defined decisioning states and failure modes with recovery paths
  • Standardised patterns for evidence capture, review, and escalation

User Journey Mapping

Created detailed journey maps identifying pain points and opportunities:

  • Mapped complete journey from awareness to account activation
  • Identified 8 key decision points requiring user action
  • Designed recovery paths for each failure mode

Design & Iteration

Early Concepts

Explored multiple approaches before converging on the final solution:

  • Tested progressive disclosure vs. upfront information architecture
  • Explored different status communication patterns
  • Validated recovery path designs through user testing

Key Design Decisions

Progressive Status Communication
Implemented real-time status updates with clear explanations. Users always know where they are and what's happening, reducing anxiety and support tickets.
Smart Document Capture
Added inline validation and guidance during document upload, reducing rejection rates by 40%.
Recovery-First Error Handling
Every error state includes specific guidance and a clear path forward, not just an error message.
Audit Trail Transparency
Users can see their verification history, building trust and reducing confusion about requirements.

Design System

Created a scalable design system ensuring consistency and compliance:

  • Component library with 40+ reusable components
  • Token system mapping directly to code (colors, spacing, typography)
  • Accessibility guidelines embedded in every component
  • Pattern library for verification flows, status communication, and error handling
